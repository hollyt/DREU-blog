Week 7 - 07.07.14 - 07.11.14

I spent this week continuing to train and test my classifier in order to make it as accurate as possible. We decided to split the data set up into data for 24 individual people rather than 12 dyads.

The reasoning behind this change is that when the classifier is implemented in our virtual human and predicts friendship status in real-time, it will be calculating data for itself and for the human user separately. Training the classifier by splitting the data more closely represents the context of the real-time system.

Splitting up the dyads means that there's a lot more data to deal with now - 4518 data points and 1084 features! I needed to figure out a way to easily split the data up into training and testing sets, and unfortunately Python scripting is really inefficient for that since I didn't want to have to re-write the data file every time I made a change to it.

I ended up taking advantage of the lab's server and using sed to easily and quickly manage the data set. Here's a little example of what I did to break the data up into training and testing sets:


			sed -n '1,145 p' <datafile >testfile
			sed '1,145 d' <datafile >trainingfile

For example, say we want to train our algorithm on the data for all people expect Person 1, and then test the accuracy of the algorithm on the data from Person 1. If lines 1 - 145 in the data file belong to Person 1, these sed commands create a training file with all data except Person 1's data and creates a testing file with Person 1's data.
